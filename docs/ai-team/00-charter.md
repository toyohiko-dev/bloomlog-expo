# BloomLog 開発憲章

## プロダクト定義
BloomLogは、大規模イベントの体験を「1日単位で記録し、あとから振り返る」ためのアプリである。
情報アプリではなく、体験ログアプリである。

## コア体験
1. 来場日（セッション）を作る
2. 体験をすぐ記録できる
3. 後から見返して楽しい

## 機能軸
すべての記録は以下の4軸に収まる
- パビリオン訪問
- フード
- ピン
- イベント参加

## UX原則
- 迷わせない
- 1画面1目的
- 入力は最短導線
- 初見で理解できる
- 空状態でも意味が伝わる

## 文言ルール
- 日本語で簡潔に
- 動詞中心
- 用語を統一する

## データ設計原則
- セッションが最上位
- UIと一致する構造
- 過剰設計しない

## 技術方針
- Next.js + Supabase + Vercel
- 小さく作って伸ばす
- Server Actions中心

## 優先順位
最優先：
- 記録しやすさ
- ホームのわかりやすさ
- 導線の短さ

低優先：
- SNS
- 通知
- レコメンド
